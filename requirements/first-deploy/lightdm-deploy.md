Контекст

Упростить деплой графических конфигураций — в первую очередь `~/.xinitrc` и конфиги/скрипты для LightDM — с упором на управление через `chezmoi` (примерно 70% через `chezmoi`, 30% — код/скрипты там, где требуется генерация).

Краткие итоги

- В репозитории есть `dot_xinitrc` и дерево `scripts/dotfiles/etc/...`.
- Основная стратегия: хранить файлы в `chezmoi` source — домашние файлы как `dot_...`, системные как `etc/...`.
- В вашем случае LightDM‑файлы располагаются в `/etc/lightdm`.
- Git/Windows не хранит UNIX‑владельца/группу; owner/group и окончательные права задаются на целевой Linux‑машине (однократно при seed‑импорте или автоматизированно через bootstrap/CI/Ansible).

Цели

- Сохранить репозиторий ОС‑агностичным.
- Обеспечить корректный `mode` и `owner:group` при деплое (`lightdm:lightdm` для LightDM‑скриптов) без ручной правки на каждой машине.
- Предпочтительный подход: единоразовый seed‑import на эталонной Linux‑машине (или в CI), `sudo chezmoi add ...` — сохранить метаданные в source; далее на остальных хостах выполнить `sudo chezmoi apply`.

Коротко про права и владельцев

- Репозиторий остаётся ОС‑агностичным; конечные права и owner/group устанавливаются на целевой машине.
- Git на Windows можно пометить execute‑бит (`git update-index --chmod=+x ...`), но owner/group сохраняются только после захвата на Linux (`chezmoi add` или ручной `chown`+`chmod`).
