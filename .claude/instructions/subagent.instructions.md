# Subagent Instructions (Claude Code)

Доступные субагенты находятся в `.claude/agents/`. Субагенты не могут вызывать другие субагенты — chaining выполняется из основного разговора.

## Доступные субагенты

| Агент | Назначение | Модель |
|-------|-----------|--------|
| `reader` | Сбор контекста из файлов и внешней документации | haiku |
| `linter` | Запуск линтеров, тестов, статических проверок | haiku |
| `fixer` | Исправление ошибок в коде | sonnet |
| `claudette` | Автономное решение сложных многошаговых задач (enterprise agent) | opus |

## Рабочий процесс (chaining из основного разговора)

Для задач, требующих исследования → проверки → исправления, используй субагентов последовательно:

1. Делегируй `reader` для сбора контекста по указанным файлам/паттернам.
2. Делегируй `linter` для запуска проверок и сбора ошибок.
3. Если linter нашёл ошибки — делегируй `fixer` с описанием найденных проблем.
4. При необходимости повтори цикл linter → fixer до прохождения всех проверок.

Для автономных сложных задач делегируй `claudette`, который самостоятельно исследует, реализует и тестирует решение.

## Практические указания

- Каждый субагент работает в изолированном контексте и возвращает только результат.
- Передавай субагенту конкретное задание: файлы, паттерны, команды, описание цели.
- Субагенты записывают находки и открытия в MEMORY (auto memory). При противоречии с существующей информацией — старую помечать "⚠️ Старая информация, вернуться позже", не удалять.
- Не выполняй `git commit`/`push` автоматически — покажи пользователю команды.
- При работе с внешними зависимостями используй WebFetch для проверки актуальности.
